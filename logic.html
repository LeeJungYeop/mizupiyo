00 한 줄로

발표의 중심 메시지. 나머지 슬라이드는 전부 이 문장을 풀어 설명하는 것.

문제 — 더운 날일수록 위험한데도, 사람들은 수분 섭취를 미룬다.

접근(넛지) — 경고·알림이 아니라 "돌보고 싶은 재미"로 유도한다. 실제 삿포로 기온에 반응해 목말라하는 오목눈이(시마에나가)에게 물을 주며 키우면, 사용자가 자기도 물을 마시게 된다.

실제 날씨 데이터가 게임 난이도와 경고 문구를 동시에 움직인다.
덥다 → 펫이 더 빨리 목마르고 → 더 자주 챙겨야 하고 → 넛지 숫자도 커진다.

01 쓰는 데이터 3종

두 가지 성격 — 실시간 오늘을 판단하는 데이터, 과거 10년 경고에 무게를 싣는 데이터.

실시간 · API

Open-Meteo 기온

지금 내 위치(또는 삿포로)의 오늘 최고기온. 무료·키 불필요·설치 없음.

과거 10년 · CSV

삿포로 일별 기상

10년치 날짜별 최고기온. 과거 각 날의 더위 등급을 판정하는 기준.

과거 10년 · CSV

홋카이도 열사병 이송

10년치 날짜별 열사병 이송자 수. 넛지 문구의 숫자 근거.

02 핵심 로직 A — 날씨가 난이도를 정한다

실시간 최고기온→ 오늘 더위 단계→ 게임 난이도 3요소

오늘 최고기온을 4단계로 나누고, 그 단계가 게임의 세 가지를 동시에 조절한다. 더울수록 현실에서 더 자주 마셔야 하는 것과 똑같이 맞춰진다.

더위 단계기온(최고)목마름 속도물 주는 간격한 잔 보상
쾌적< 25℃느림길다작다
더움25–29℃보통보통보통
真夏日 (한여름)30–34℃빠름짧다크다
猛暑日 (폭염)≥ 35℃매우 빠름가장 짧다가장 크다

한 줄: 더운 날일수록 더 자주 챙겨야 하고, 챙겼을 때 보상도 크다 — 게임 규칙이 곧 건강 메시지.

03 핵심 로직 B — 과거 데이터가 경고에 힘을 싣는다

두 CSV를 날짜로 조인→ 단계별 역대 평균 이송자→ 오늘 단계의 숫자를 문구로

열사병 데이터엔 "그날 몇 명 이송"만 있고 그날이 더웠는지는 없다. 그래서 기상 데이터와 같은 날짜끼리 붙여 각 날에 기온을 달아주고, 더위 단계로 묶어 단계별 하루 평균을 미리 계산해 둔다.

과거 날짜기상 CSV: 기온→ 단계열사병 CSV: 이송
2018-07-2033℃真夏日45명
2019-08-0232℃真夏日57명
2018-07-2136℃猛暑日80명
  • 真夏日 칸에 모인 날들의 평균 → 약 51명
  • 오늘이 真夏日이면 그 숫자를 꺼내 문구로: "真夏日엔 홋카이도에서 하루 평균 51명이 열사병으로 이송됐어요"
  • 같은 뜻을 날마다 다른 표현으로 로테이션(같은 날은 고정) — 매일 봐도 지겹지 않게

04 육성 루프 — 실제로 물을 마시게

두 로직이 만든 난이도·문구 위에서 실제 플레이가 돈다.

  • 물 마시기 = 1잔 — 화면의 물주기가 곧 "내가 실제로 한 잔 마셨다". 하루 목표 8잔.
  • 수분 게이지 — 시간이 지나면 줄고(단계별 속도), 한 잔 마시면 회복. 낮아지면 펫이 목말라한다.
  • 펫 반응 — 평온 / 목마름 / 폭염지침, 마시면 꿀꺽 → 기뻐함, 탭하면 쓰다듬기 반응. 픽셀 도트 오목눈이.
  • 연속 기록 — 며칠째 꾸준히 챙겼는지. 하루라도 놓치면 리셋.
  • 저장 — 모든 상태는 브라우저(localStorage)에. 로그인·서버 없음.
